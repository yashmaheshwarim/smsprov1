# Timetable Page Enhancement Plan

## Problem Statement
The Timetable section needed to load or display the Batch Management section batches in a list to manage the timetable of that batch. The previous implementation had syntax errors and used hardcoded batch data.

## Issues Fixed
1. **Syntax Error**: Duplicate import of `useAuth` causing "Identifier 'useAuth' has already been declared"
2. **React Warning**: `value` prop on select was null when no batches were loaded
3. **Object Rendering Error**: Attempting to render batch objects directly as options instead of extracting string values

## Solution Implemented

### Key Changes Made to `src/pages/TimetablePage.tsx`:

#### 1. Fixed Imports
- Removed duplicate `useAuth` import
- Cleaned up duplicate `toast` imports
- Organized import statements properly

#### 2. Enhanced Batch Loading
- Replaced hardcoded batches array with dynamic loading from Supabase `batches` table
- Added `fetchBatches()` function that:
  - Fetches batches for the current institute from Supabase
  - Sets default selected batch to first batch if available
  - Handles loading and error states

#### 3. Integrated Timetable with Database
- Added `fetchTimetable()` function to load timetable entries from Supabase `timetable` table
- Implemented CRUD operations for timetable entries:
  - `saveTimetableEntry()` - Create new entries
  - `updateTimetableEntry()` - Modify existing entries
  - `deleteTimetableEntry()` - Remove entries
- All operations include proper error handling and user feedback via toast notifications

#### 4. Fixed Select Component Issues
- Modified batch select to handle null/undefined values properly:
  - Show "Loading batches..." option while fetching
  - Use batch ID as option key for uniqueness
  - Extract batch name for option value and display
  - Properly handle batch selection changes with object lookup
- Added conditional rendering based on whether batches are loaded

#### 5. Enhanced Data Model
- Updated `TimetableEntry` interface to include `instituteId` for proper data isolation
- Added `Batch` interface for type safety

#### 6. Maintained Existing UI Functionality
- Preserved desktop grid view of timetable
- Maintained mobile list view
- Kept add/edit dialog functionality
- Retained visual subject coloring
- Preserved time slot selection and day navigation

## Technical Details

### Database Schema Assumptions
- `batches` table with columns: `id`, `name`, `institute_id`, `created_at`
- `timetable` table with columns matching `TimetableEntry` interface:
  - `id`, `day`, `startTime`, `endTime`, `subject`, `teacher`, `room`, `batch`, `instituteId`

### Data Flow
1. Component mounts → Fetch batches for current institute
2. On batch selection change → Fetch timetable entries for selected batch
3. User actions (add/edit/delete) → Perform Supabase operation → Refresh timetable list
4. All operations are scoped to current institute for multi-tenancy

### User Experience
- Dropdown shows all available batches from database
- Default selection is first batch (if any exist)
- Selecting a batch loads its timetable entries
- Add/edit dialog allows creating/modifying timetable slots
- Visual feedback for all operations (toasts, loading states)
- Responsive design works on both desktop and mobile
