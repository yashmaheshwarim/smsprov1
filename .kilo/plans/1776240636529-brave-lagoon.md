# Plan: Fix Production Blank Screen Issue

## Problem Summary
The app works on localhost but shows blank in production. This is a common issue where environment variables aren't properly embedded in Vite production builds.

## Root Causes Identified

### 1. Environment Variables Not Embedded (Primary Issue)
**Location:** `src/lib/supabase.ts:3-9`

The Supabase client is initialized with fallback values, but if env vars are missing in production:
```typescript
const supabaseUrl = import.meta.env.VITE_SUPABASE_URL || ...;
const supabaseAnonKey = import.meta.env.VITE_SUPABASE_ANON_KEY || ...;
export const supabase = createClient(supabaseUrl, supabaseAnonKey);
```

Vite embeds env vars at **build time**. If `.env` isn't properly loaded during production build, these become undefined, causing all Supabase queries to fail silently.

### 2. No Error Boundaries
The app has no error boundaries to catch render failures or display meaningful error UI when data fetching fails.

### 3. Silent Query Failures
All data-fetching pages use Supabase queries that silently return empty arrays on failure instead of showing errors.

---

## Solution Plan

### Step 1: Add Build-Time Env Validation
Modify `src/lib/supabase.ts` to throw clear errors if env vars are missing:
- Check for undefined env vars before creating client
- Throw descriptive error that's visible in console
- Add debugging logs for production troubleshooting

### Step 2: Add Error Boundary
Create an error boundary component:
- Wrap the app in `src/App.tsx`
- Show fallback UI when errors occur
- Display helpful error message with troubleshooting steps

### Step 3: Add Loading/Error States to Pages
Update key pages to show appropriate states:
- DashboardPage (most visible)
- LoginPage redirect handling

### Step 4: Verify Build Configuration
Check/fix Vite config for proper env handling:
- Ensure `.env` is included in build process
- Or add explicit env passing in build script
