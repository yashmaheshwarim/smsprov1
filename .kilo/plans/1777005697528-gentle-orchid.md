# Plan: Split Fees Management into Two Pages

## Current State
- Single `FeesManagementPage.tsx` with a `viewMode` toggle between "batch" and "student"
- Route: `/fees` → `FeesManagementPage`
- Sidebar: Single "Fees" link pointing to `/fees`

## Goal
Split into two separate pages with distinct responsibilities:

### 1. Batch Fee Page (`/fees/batch`)
- **Purpose:** Configure fee structures at the batch level
- **Functionality:**
  - Create batch fee records (title, total fees, description, due date, batch selection)
  - Edit existing batch fees
  - Delete batch fees (with cascade delete of related student fee records)
  - **Auto-apply:** When a batch fee is created, it automatically creates `student_fees` records for ALL active students in that batch
  - Shows batch fee list with columns: Fee Title, Total Fee, Student Count, Due Date, Created, Actions
- **No stats cards needed** - this is an admin/configuration page

### 2. Student Fee Page (`/fees/student`)
- **Purpose:** View and manage individual student fee records
- **Functionality:**
  - **Data source:** Displays all `student_fees` records that were auto-created from batch fees
  - Each student fee row shows: Student Name, Enrollment, Batch, Original Fee, Discount, Final Fee, Paid, Pending, Status, Actions
  - Stats cards at top: Total Fees, Collected, Pending, Overdue (calculated from student_fees)
  - Search by student name/enrollment/id
  - Filter by status (all, paid, pending, partial, overdue)
  - Pagination support
  - Actions:
    - **Pay:** Record payment, update paid_fees and status
    - **Discount:** Apply discount amount, update final fee
    - **Receipt:** Generate PDF receipt (using jsPDF)
- This page is read-only for the fee amounts (original fee comes from batch), but allows payments/discounts

## Database Flow
```
Batch (e.g., "JEE 2026 Batch A")
  └── BatchFee record created (title, total_fees, batch_id)
       └── Auto-creates StudentFee records for each student in the batch
            └── StudentFee: original_fee = batch_fee.total_fees
                 └── Payments & discounts tracked here
```
